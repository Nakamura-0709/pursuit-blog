# Git ワークフロー

## コミット

- **Conventional Commits** に従う（`feat:`, `fix:`, `chore:`, `docs:`, `ci:`, `refactor:`）
- 1 コミット = 1 論理的な変更（アトミックコミット）
- コミットメッセージは「なぜ」に焦点を当て、変更の目的を正確に反映する
- 可能な場合は `git commit -am` を使用する
- 関係ないファイルは含めない
- git 設定は変更しない
- 対話的な git コマンド（`-i` フラグ）は使用しない

## プルリクエスト

- PR 作成時は必ず `-a @me` を付けて自分をアサインする
- 必要に応じて新しいブランチを作成し、`-u` フラグでリモートへプッシュする

```bash
gh pr create -a @me --title "feat: 変更の概要" --body "$(cat <<'EOF'
## 概要
- 変更の目的と背景

## 変更内容
- 具体的な変更点

## レビューのポイント
- レビュー時に注目してほしい点
EOF
)"
```

## PR マージ

- `gh pr merge -d` でマージし、head ブランチをリモート・ローカルから削除する
- マージ後にリモートで削除済みの参照を整理する: `git fetch --prune`

## ブランチ命名

- 機能追加・インフラ追加: `feat/xxx`
- バグ修正・設定修正: `fix/xxx`
- ドキュメント: `docs/xxx`
