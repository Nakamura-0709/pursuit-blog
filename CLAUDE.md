# CLAUDE.md

このファイルはプロジェクト全体の意思決定基準と絶対的な制約（Guardrails）を定義する。
コーディング規約・Git 運用などの詳細は `.claude/rules/` 配下のルールファイルに従うこと。

## 技術選定

- **唯一の正**: [docs/01_tech-stack.md](docs/01_tech-stack.md) に従う。未記載の技術を勝手に導入しない
- 新たに技術を採用する場合は、先に docs/01_tech-stack.md を更新してから実装する
- **開発の流れ**: [docs/02_development-flow.md](docs/02_development-flow.md) の手順に従う
- **実装フェーズ**: [docs/04_implementation-phases.md](docs/04_implementation-phases.md) を参照

## 絶対的な制約（Guardrails）

- **破壊的変更の禁止**: RDS・ECS・DynamoDB・S3 バケットなど本番リソースの削除や変更を伴う提案時は、必ずユーザーに二重確認を行うこと
- **テストの失敗**: 2回以上連続でテストを失敗した時は、独断で進めず現在の状況を整理してユーザーと一緒に解決方法を考える
- **認証情報**: Access Key やパスワードなどはハードコードせず、環境変数やシークレット管理を利用する
- **セキュリティ**: IAM 権限の最小化、ポート開放の制限などセキュリティベストプラクティスを考慮する

## 作業開始準備

- `git status` で現在の git コンテキストを確認する
- 指示された内容と無関係な変更が多い場合、別タスクとして開始するように提案する
- コンテキストが不明瞭な時はユーザーに確認する
